# ΠΛΑΝΟ ΥΛΟΠΟΙΗΣΗΣ
## Έξυπνο Εργαλείο Ανάλυσης Ασφαλιστικών Αναγκών — NN Hellas

> Ενοποιημένο πλάνο: συγχωνεύει ΕΠΙΠΕΔΟ 1 (Μηχανή Κανόνων) και ΕΠΙΠΕΔΟ 2 (AI Layer)
> και τα αναδιοργανώνει **ανά κατηγορία πράξης**, με υποκατηγορίες για κάθε ενδιάμεσο βήμα.
>
> Αυτό το αρχείο είναι η **πηγή αλήθειας** της υλοποίησης. Διαβάζεται μαζί με το `MASTER_PROMPT_Claude_Code.md`.

---

## 0. Σκοπός & Πλαίσιο

**Τι υπάρχει σήμερα:** Ένα λειτουργικό standalone αρχείο `index.html` (~2.045 γραμμές) — διαδραστικό εργαλείο που καθοδηγεί τον πελάτη σε 6 βήματα ανάλυσης αναγκών, υπολογίζει score (Υγεία / Ζωή / Σύνταξη), παράγει 3 σενάρια πρότασης (Α/Β/Γ) και στέλνει email + καταγράφει σε Google Sheet.

**Το πρόβλημα:** Όλα σε ένα αρχείο· προϊόντα, τιμές και κανόνες διάσπαρτα μέσα στον κώδικα· ο πίνακας συμβατότητας προγραμμάτων–προσαρτημάτων δεν επιβάλλεται· κάθε αλλαγή απαιτεί παρέμβαση στον κώδικα.

**Ο στόχος:**
- **ΕΠΙΠΕΔΟ 1 — Μηχανή Κανόνων:** ανακατασκευή του «παρασκηνίου» ώστε το εργαλείο να γίνει αξιόπιστο, συντηρήσιμο και πάντα έγκυρο. Η εμπειρία του πελάτη μένει αμετάβλητη.
- **ΕΠΙΠΕΔΟ 2 — AI Layer:** προσθήκη ενός στρώματος τεχνητής νοημοσύνης που συνθέτει εξατομικευμένο κείμενο πρότασης, χωρίς να αναλαμβάνει την επιλογή προϊόντων.

**Εκτός σκοπού (μελλοντικό):** ΕΠΙΠΕΔΟ 3 — Feedback Layer (μάθηση από δεδομένα). Βλ. §6.

## 1. Αρχιτεκτονική-στόχος

Αντικατάσταση του ενιαίου `index.html` με διακριτά, ανεξάρτητα στρώματα:

| Αρχείο | Ρόλος |
|---|---|
| `data.js` | **Μοναδική πηγή αλήθειας** — προγράμματα, προσαρτήματα, τιμές, πίνακας συμβατότητας 18×13, κατώφλια scoring |
| `engine.js` | Μηχανή Κανόνων — scoring, επιλογή προϊόντων, επικύρωση, παραγωγή σεναρίων |
| `ui.js` | Παρουσίαση — ερωτηματολόγιο, οθόνες, εμφάνιση αποτελεσμάτων |
| `integrations.js` | Email & καταγραφή σε Google Sheet |
| `styles.css` | Εμφάνιση (αμετάβλητη οπτικά) |
| `index.html` | Σκελετός που ενώνει τα παραπάνω |
| `backend proxy` | (ΕΠ.2) serverless function — κρατά ασφαλές το API key |
| `system prompt` | (ΕΠ.2) versioned αρχείο με τις οδηγίες του AI |

**Δύο στρώματα ευφυΐας:**
1. **Rules Engine (ΕΠ.1):** αποφασίζει *ποια* προϊόντα — ντετερμινιστικά, βάσει κανόνων.
2. **AI Layer (ΕΠ.2):** αποφασίζει *πώς επικοινωνείται* η πρόταση — εξατομικευμένα. **Δεν** επιλέγει ποτέ προϊόντα.

## 2. Κατηγορίες Πράξης

Επτά κατηγορίες (Α–Ζ). Κάθε μία περιλαμβάνει: **Σκοπό**, **Υποκατηγορίες** (με τα ενδιάμεσα βήματα), **Εξαρτήσεις** και **Ορισμό Ολοκλήρωσης** (το κριτήριο για να θεωρηθεί «έτοιμη»).

---

### Α. ΠΡΟΕΤΟΙΜΑΣΙΑ & ΔΕΔΟΜΕΝΑ
*(ΕΠΙΠΕΔΟ 1 — θεμέλιο)*

**Σκοπός:** Δημιουργία της ενιαίας, αξιόπιστης πηγής δεδομένων (`data.js`).

**Υποκατηγορίες:**
- **Α1. Αντιστοίχιση ονοματολογίας προϊόντων.** Χαρτογράφηση των ονομάτων του spreadsheet έναντι της τρέχουσας εφαρμογής (π.χ. *Easy Group ↔ Easy Dunant*, *FIRST CARE ↔ OFC*, *CROSS ↔ Orange Cross*). Εντοπισμός προϊόντων που λείπουν από την εφαρμογή (Orange Help, Single Flex, Hospital For All, βαθμίδες MediPlan).
- **Α2. Συγκέντρωση τιμοκαταλόγων.** Όλες οι τιμές προγραμμάτων & προσαρτημάτων, ανά παράμετρο (ηλικία/πακέτο κ.λπ.).
- **Α3. Καταγραφή πίνακα συμβατότητας 18×13.** Μεταφορά του πίνακα από το spreadsheet σε δομημένη μορφή matrix (18 προγράμματα × 13 προσαρτήματα, ομαδοποιημένα σε 6 κατηγορίες προσαρτημάτων).
- **Α4. Σχεδιασμός σχήματος του `data.js`.** Ορισμός δομής: `programs`, `attachments`, `attachmentCategories`, `prices`, `compatibilityMatrix`, `scoringThresholds`.
- **Α5. Συμπλήρωση του `data.js`.** Μεταφορά του 100% των δεδομένων στο σχήμα — καμία διάσπαρτη σταθερά οπουδήποτε αλλού.

**Εξαρτήσεις:** Καμία. Είναι το πρώτο βήμα.

**Ορισμός Ολοκλήρωσης:** Υπάρχει ένα `data.js` που περιέχει όλα τα προϊόντα, τις τιμές και τους κανόνες συμβατότητας. Κανένα δεδομένο δεν είναι «καρφωτό» αλλού στον κώδικα.

---

### Β. ΜΗΧΑΝΗ ΚΑΝΟΝΩΝ
*(ΕΠΙΠΕΔΟ 1 — πυρήνας)*

**Σκοπός:** Μηχανή που λαμβάνει αποφάσεις διαβάζοντας το `data.js` — διαφανής και προβλέψιμη.

**Υποκατηγορίες:**
- **Β1. Scoring.** Υπολογισμός αναγκών Υγείας / Ζωής / Σύνταξης από τις απαντήσεις του ερωτηματολογίου, βάσει των κατωφλίων του `data.js`.
- **Β2. Επιλογή προγραμμάτων.** Ένα βασικό πρόγραμμα ανά οικογένεια (Ζωής / Υγείας / Αποταμίευσης), βάσει score και προϋπολογισμού.
- **Β3. Μηχανισμός συμβατότητας.** Φιλτράρισμα των έγκυρων προσαρτημάτων με βάση τον πίνακα 18×13 — κανόνας: το πολύ 1 προσάρτημα ανά κατηγορία, επιτρεπτοί συνδυασμοί μεταξύ κατηγοριών.
- **Β4. Παραγωγή σεναρίων.** Σύνθεση 3 προτάσεων: Προτεινόμενο, Βασικό, Εκτεταμένο.
- **Β5. Επικύρωση εξόδου.** Έλεγχος ότι κάθε παραγόμενη πρόταση τηρεί όλους τους κανόνες — **καμία μη έγκυρη πρόταση δεν φτάνει στον χρήστη**.
- **Β6. Διαχωρισμός κώδικα.** Το `engine.js` είναι καθαρό — μόνο λογική, χωρίς UI και χωρίς ενσωματωμένα δεδομένα.

**Εξαρτήσεις:** Α (απαιτείται το `data.js`).

**Ορισμός Ολοκλήρωσης:** Το `engine.js` παράγει μόνο έγκυρους συνδυασμούς για οποιοδήποτε προφίλ· επαληθεύεται με ένα σύνολο δοκιμαστικών προφίλ.

---

### Γ. FRONTEND / UI
*(ΕΠΙΠΕΔΟ 1)*

**Σκοπός:** Σύνδεση του υπάρχοντος UI με τη νέα μηχανή — **χωρίς καμία αλλαγή στην εμπειρία του πελάτη**.

**Υποκατηγορίες:**
- **Γ1. Απομόνωση UI σε `ui.js`.** Διαχωρισμός της παρουσίασης από τη λογική.
- **Γ2. Αναθεώρηση ερωτηματολογίου.** Επιβεβαίωση ότι οι 6 ερωτήσεις τροφοδοτούν σωστά τη μηχανή· προσθήκη όποιων στοιχείων χρειάζεται το νέο scoring.
- **Γ3. Σύνδεση UI ↔ engine.** Το UI καλεί το `engine.js` και εμφανίζει τα 3 σενάρια στην ίδια οθόνη αποτελεσμάτων.
- **Γ4. Διατήρηση εμφάνισης.** Το `styles.css` απομονώνεται· η οπτική παραμένει πανομοιότυπη.

**Εξαρτήσεις:** Β.

**Ορισμός Ολοκλήρωσης:** Το εργαλείο λειτουργεί end-to-end με τη νέα μηχανή, με ίδια ακριβώς εμπειρία χρήστη.

---

### Δ. BACKEND & AI LAYER
*(ΕΠΙΠΕΔΟ 2)*

**Σκοπός:** Προσθήκη του στρώματος AI που συνθέτει εξατομικευμένο κείμενο πρότασης.

**Υποκατηγορίες:**
- **Δ1. Backend proxy.** Serverless function (Cloudflare Workers ή Google Apps Script) που κρατά το API key και προωθεί τα αιτήματα, με rate limiting.
- **Δ2. System prompt.** Σχεδιασμός και δοκιμή του prompt: γνώση προϊόντων NN, ύφος (ανθρώπινο, ενσυναισθητικό, στον ενικό), αυστηρά όρια, μορφή εξόδου. Versioned αρχείο.
- **Δ3. Ενσωμάτωση κλήσης AI.** Στο βήμα αποτελεσμάτων, αντικατάσταση του template generator με κλήση στο Claude API (μέσω του proxy).
- **Δ4. Fallback.** Αν η κλήση αποτύχει ή λήξει ο χρόνος → αυτόματη επιστροφή στο κείμενο-template του ΕΠΙΠΕΔΟΥ 1. Ποτέ «σπασμένη» οθόνη.
- **Δ5. Guardrails & disclaimers.** Επικύρωση της εξόδου του AI, υποχρεωτικά disclaimers, ελαχιστοποίηση δεδομένων που στέλνονται (GDPR).

**Εξαρτήσεις:** Γ (το εργαλείο πρέπει πρώτα να λειτουργεί πλήρως με το ΕΠΙΠΕΔΟ 1).

**Ορισμός Ολοκλήρωσης:** Το βήμα αποτελεσμάτων παράγει εξατομικευμένο κείμενο μέσω AI, με ασφαλώς αποθηκευμένο key και λειτουργικό fallback. Το AI δεν επιλέγει ποτέ προϊόντα.

---

### Ε. ΕΝΣΩΜΑΤΩΣΕΙΣ
*(ΕΠΙΠΕΔΟ 1)*

**Σκοπός:** Αξιόπιστη αποστολή email και καταγραφή δεδομένων.

**Υποκατηγορίες:**
- **Ε1. Απομόνωση σε `integrations.js`.** Email & Sheets σε ξεχωριστό module.
- **Ε2. Email.** Επιβεβαίωση αποστολής (EmailJS ή εναλλακτική μέσω Apps Script).
- **Ε3. Καταγραφή σε Google Sheets.** Δομημένη εγγραφή μέσω του Apps Script endpoint.

**Εξαρτήσεις:** Γ. *(Μπορεί να εκτελεστεί παράλληλα με την Γ.)*

**Ορισμός Ολοκλήρωσης:** Email και καταγραφή λειτουργούν αξιόπιστα, πλήρως απομονωμένα από την υπόλοιπη λογική.

---

### ΣΤ. ΔΟΚΙΜΕΣ & ΠΟΙΟΤΗΤΑ
*(διατρέχει όλα τα επίπεδα)*

**Σκοπός:** Επιβεβαίωση ορθότητας, ασφάλειας και κανονιστικής συμμόρφωσης.

**Υποκατηγορίες:**
- **ΣΤ1. Λειτουργικός έλεγχος ανά module.** `data`, `engine`, `ui`, `integrations` — το καθένα ξεχωριστά.
- **ΣΤ2. Δοκιμαστικά προφίλ.** Σύνολο σεναρίων πελατών με αναμενόμενα αποτελέσματα, για επαλήθευση της μηχανής.
- **ΣΤ3. Cross-browser & mobile.** Chrome, Firefox, Safari, Edge, κινητά.
- **ΣΤ4. Έλεγχος AI (ΕΠ.2).** Ποιότητα & συνέπεια κειμένου, latency, πραγματικό κόστος ανά κλήση.
- **ΣΤ5. GDPR & συμμόρφωση.** Ελαχιστοποίηση δεδομένων, παρουσία disclaimers, καμία μη έγκυρη πρόταση.

**Εξαρτήσεις:** Εκτελείται μετά/παράλληλα με κάθε κατηγορία — όχι στο τέλος μόνο.

**Ορισμός Ολοκλήρωσης:** Όλα τα δοκιμαστικά προφίλ περνούν· καμία μη έγκυρη πρόταση· κόστος AI εντός εκτίμησης.

---

### Ζ. ΠΑΡΑΔΟΣΗ & ΛΕΙΤΟΥΡΓΙΑ

**Σκοπός:** Θέση σε λειτουργία και διασφάλιση βιωσιμότητας.

**Υποκατηγορίες:**
- **Ζ1. Deployment.** Frontend σε static hosting· backend proxy σε serverless πλατφόρμα.
- **Ζ2. Τεκμηρίωση.** Σύντομος οδηγός: πώς ενημερώνεται το `data.js`, πώς τροποποιείται το system prompt.
- **Ζ3. Διαδικασία ενημέρωσης.** Σαφή βήματα για αλλαγή τιμής ή προσθήκη προϊόντος — όλα σε ένα σημείο.
- **Ζ4. Παρακολούθηση κόστους.** Ειδοποίηση προϋπολογισμού (budget alert) για το AI API.

**Εξαρτήσεις:** Όλες οι προηγούμενες.

**Ορισμός Ολοκλήρωσης:** Το εργαλείο είναι σε λειτουργία, τεκμηριωμένο, με ξεκάθαρη διαδικασία συντήρησης.

## 3. Σειρά Εκτέλεσης & Εξαρτήσεις

```
Α  →  Β  →  Γ  →  Δ  →  Ζ
              ↘  Ε  ↗
        ΣΤ (συνεχώς, σε κάθε στάδιο)
```

- **ΕΠΙΠΕΔΟ 1** ολοκληρώνεται με: Α + Β + Γ + Ε (+ ΣΤ μερικώς + Ζ μερικώς).
- **ΕΠΙΠΕΔΟ 2** προστίθεται με: Δ (+ ΣΤ4 + Ζ για το backend).
- Καμία κατηγορία δεν ξεκινά πριν ολοκληρωθούν οι εξαρτήσεις της. Η Ε μπορεί να τρέξει παράλληλα με τη Γ.

## 5. Κίνδυνοι & Αντιμετώπιση

| Κίνδυνος | Αντιμετώπιση |
|---|---|
| Ασυμφωνία ονομάτων προϊόντων spreadsheet ↔ εφαρμογή | Κατηγορία Α1 — ρητή χαρτογράφηση πριν από οτιδήποτε άλλο |
| Παραγωγή μη έγκυρης πρότασης | Β5 — υποχρεωτική επικύρωση εξόδου με τον πίνακα 18×13 |
| Διαρροή / κατάχρηση API key | Δ1 — το key μόνο στο backend proxy, ποτέ στον client |
| Διακοπή ή καθυστέρηση του AI | Δ4 — αυτόματο fallback στο template του ΕΠ.1 |
| Το AI «εφευρίσκει» προϊόν ή τιμή | Το prompt δίνει μόνο τα έγκυρα προϊόντα του engine· έλεγχος εξόδου (Δ5) |
| Ανησυχία για GDPR | Δ5 — ελαχιστοποίηση δεδομένων· το εμπορικό API δεν εκπαιδεύεται στα δεδομένα |
| Κόστος AI εκτός ελέγχου | Ζ4 — όρια `max_tokens`, prompt caching, budget alert |

## 6. Εκτός Σκοπού — ΕΠΙΠΕΔΟ 3 (Μελλοντικό)

Το **Feedback Layer**: καταγραφή του τι έγινε με κάθε πρόταση (αποδοχή / τροποποίηση / απόρριψη) και περιοδική ανάλυση που τροφοδοτεί τη βελτίωση των κανόνων. Είναι το σημείο όπου το σύστημα αρχίζει πραγματικά να «μαθαίνει». Χτίζεται πάνω στην ίδια βάση, **χωρίς ξαναγράψιμο** — δεν περιλαμβάνεται σε αυτό το πλάνο.
